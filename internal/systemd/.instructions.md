# Systemd Integration Instructions

**Scope:** `internal/systemd/**/*.go`

These instructions apply to all systemd-related code in the rclone-mount-sync application.

---

## Architecture Overview

The systemd integration consists of three main components:

1. **Generator** (`generator.go`) - Creates unit files from templates
2. **Manager** (`manager.go`) - Controls service lifecycle (start/stop/enable/disable)
3. **Reconciler** (`reconcile.go`) - Handles orphan unit cleanup

### Component Responsibilities

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  Generator  │─────▶│   Manager   │─────▶│ Reconciler  │
│  (Create)   │      │  (Control)  │      │  (Cleanup)  │
└─────────────┘      └─────────────┘      └─────────────┘
```

## Unit File Templates

### Available Templates

Located in `templates.go`:

1. **MountServiceTemplate** - For mount operations
   - Type: `notify` (rclone signals readiness)
   - Includes VFS cache pre-mount setup
   - Handles graceful unmount on stop

2. **SyncServiceTemplate** - For sync operations
   - Type: `oneshot` (runs once and exits)
   - Supports conditional execution (AC power, unmetered)
   - Resource limits (MemoryMax, CPUQuota)

3. **SyncTimerTemplate** - For scheduled sync
   - Configurable via `TimerDirectives`
   - Supports OnCalendar, OnBootSec, OnUnitActiveSec

### Template Variables

```go
// Mount unit data
type MountUnitData struct {
    Name         string   // Display name
    Remote       string   // Remote name (e.g., "gdrive:")
    RemotePath   string   // Remote path (e.g., "/backup")
    MountPoint   string   // Local mount point
    MountOptions string   // Combined mount flags
    RclonePath   string   // Full path to rclone binary
    LogPath      string   // Log file location
}

// Sync unit data
type SyncUnitData struct {
    Name             string   // Display name
    Source           string   // Source path
    Destination      string   // Destination path
    Direction        string   // sync/copy/move
    SyncOptions      string   // Combined sync flags
    RequireACPower   bool     // Only run on AC power
    RequireUnmetered bool     // Only run on unmetered connection
}
```

## Generator Patterns

### Creating Unit Files

```go
// ✅ CORRECT - Use generator methods
generator, err := systemd.NewGenerator()
if err != nil {
    return fmt.Errorf("failed to create generator: %w", err)
}

// Generate and write service file
servicePath, err := generator.WriteMountService(mountConfig)
if err != nil {
    return fmt.Errorf("failed to write service: %w", err)
}

// Generate timer file (for sync jobs)
timerPath, err := generator.WriteSyncTimer(syncConfig)
if err != nil {
    return fmt.Errorf("failed to write timer: %w", err)
}
```

### Building Options Strings

```go
// Use builder pattern for options
func (g *Generator) buildMountOptions(opts *MountOptions) string {
    var options []string
    
    // VFS options
    if opts.VFSCacheMode != "off" {
        options = append(options, fmt.Sprintf("--vfs-cache-mode=%s", opts.VFSCacheMode))
    }
    if opts.VFSCacheMaxSize != "" {
        options = append(options, fmt.Sprintf("--vfs-cache-max-size=%s", opts.VFSCacheMaxSize))
    }
    
    // FUSE options
    if opts.AllowOther {
        options = append(options, "--allow-other")
    }
    if opts.Umask != 0 {
        options = append(options, fmt.Sprintf("--umask=%04o", opts.Umask))
    }
    
    return strings.Join(options, " \\\n    ")
}
```

## Manager Patterns

### Service Lifecycle

```go
manager, err := systemd.NewManager()
if err != nil {
    return fmt.Errorf("failed to create manager: %w", err)
}

// Enable and start service
if err := manager.Enable(serviceName); err != nil {
    return fmt.Errorf("failed to enable: %w", err)
}

if err := manager.Start(serviceName); err != nil {
    return fmt.Errorf("failed to start: %w", err)
}

// Stop and disable service
if err := manager.Stop(serviceName); err != nil {
    return fmt.Errorf("failed to stop: %w", err)
}

if err := manager.Disable(serviceName); err != nil {
    return fmt.Errorf("failed to disable: %w", err)
}

// Reset failed state (allows restart after failure)
if err := manager.ResetFailed(serviceName); err != nil {
    return fmt.Errorf("failed to reset failed: %w", err)
}
```

### Timer Operations

```go
// Timer-specific operations
if err := manager.StartTimer(timerName); err != nil {
    return fmt.Errorf("failed to start timer: %w", err)
}

if err := manager.StopTimer(timerName); err != nil {
    return fmt.Errorf("failed to stop timer: %w", err)
}

if err := manager.EnableTimer(timerName); err != nil {
    return fmt.Errorf("failed to enable timer: %w", err)
}

if err := manager.DisableTimer(timerName); err != nil {
    return fmt.Errorf("failed to disable timer: %w", err)
}
```

### Checking Service Status

```go
// Get service status
status, err := manager.Status(serviceName)
if err != nil {
    return fmt.Errorf("failed to get status: %w", err)
}

// Check if active
if status.IsActive {
    fmt.Println("Service is running")
}

// Check if enabled
if status.IsEnabled {
    fmt.Println("Service will start on boot")
}

// Check for failures
if status.Failed {
    fmt.Println("Service has failed")
}
```

## Reconciliation Patterns

### Handling Orphan Units

When configs are deleted, orphan unit files may remain:

```go
// Detect orphan units
result, err := systemd.ReconcileUnits(config)
if err != nil {
    return fmt.Errorf("reconciliation failed: %w", err)
}

// Process orphans
for _, orphan := range result.OrphanServices {
    fmt.Printf("Found orphan service: %s\n", orphan)
    
    // Option 1: Automatically remove
    if err := manager.RemoveUnit(orphan); err != nil {
        return fmt.Errorf("failed to remove orphan: %w", err)
    }
    
    // Option 2: Prompt user (in TUI)
    // Show in UI and let user decide
}
```

### Reconciliation Result Structure

```go
type ReconciliationResult struct {
    OrphanServices []string  // Services without config
    OrphanTimers   []string  // Timers without config
    MissingConfigs []string  // Configs without units
}
```

## Path Utilities

### Getting Systemd Paths

```go
// Get user systemd directory
userDir, err := systemd.GetUserSystemdPath()
// Returns: ~/.config/systemd/user/

// Get system systemd directory (requires root)
systemDir, err := systemd.GetSystemSystemdPath()

```

## Error Handling

### Command Execution Errors

```go
// ✅ CORRECT - Check all systemctl commands
output, err := exec.Command("systemctl", "--user", "start", serviceName).CombinedOutput()
if err != nil {
    return fmt.Errorf("failed to start service: %w, output: %s", err, string(output))
}

// ❌ WRONG - Ignoring errors
exec.Command("systemctl", "--user", "start", serviceName).Run()  // Error ignored!
```

### Unit File Write Errors

```go
// ✅ CORRECT - Validate write success
err := os.WriteFile(filepath.Join(systemdDir, filename), content, 0644)
if err != nil {
    return fmt.Errorf("failed to write unit file: %w", err)
}

// Verify file exists
if _, err := os.Stat(filepath.Join(systemdDir, filename)); os.IsNotExist(err) {
    return fmt.Errorf("unit file was not created")
}
```

## Common Pitfalls

### ❌ Not Reloading Daemon After Changes

```go
// WRONG - Unit file created but daemon not notified
generator.WriteMountService(mount)
manager.Start(serviceName)  // Will fail!

// CORRECT - Reload daemon first
generator.WriteMountService(mount)
manager.DaemonReload()      // Notify systemd of changes
manager.Start(serviceName)  // Now works
```

### ❌ Ignoring Unit File Permissions

```go
// WRONG - Wrong permissions
os.WriteFile(path, content, 0777)  // Too permissive!

// CORRECT - Standard unit file permissions
os.WriteFile(path, content, 0644)  // Readable by all, writable by owner
```

### ❌ Not Handling Missing Systemd

```go
// WRONG - Assumes systemd always available
manager, _ := systemd.NewManager()  // Panics if no systemd!

// CORRECT - Check for errors
manager, err := systemd.NewManager()
if err != nil {
    return fmt.Errorf("systemd not available: %w", err)
}
```

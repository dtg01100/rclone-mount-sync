# TUI Development Instructions

**Scope:** `internal/tui/**/*.go`

These instructions apply to all TUI-related code in the rclone-mount-sync application.

---

## Architecture Overview

The TUI follows the **Elm Architecture** pattern via Bubble Tea:

```
Model → Update → View
  ↑               ↓
  └──── Msg ←─────┘
```

### Key Components

1. **App Controller** (`tui/app.go`)
   - Manages screen navigation via enum
   - Holds shared services (config, rclone, systemd)
   - Handles global keyboard shortcuts

2. **Screens** (`tui/screens/*.go`)
   - Individual views (MainMenu, Mounts, SyncJobs, etc.)
   - Each implements Bubble Tea's Model interface
   - Communicate via `ScreenChangeMsg`

3. **Components** (`tui/components/*.go`)
   - Reusable UI elements (forms, lists, dialogs)
   - Shared styling via `Styles` struct
   - Helper functions for common patterns

## Screen Navigation Pattern

### Adding a New Screen

```go
// 1. Add screen enum in tui/app.go
const (
    ScreenMain Screen = iota
    ScreenMounts
    ScreenSyncJobs
    ScreenServices
    ScreenSettings
    ScreenHelp
    ScreenYourNewScreen  // ← Add here
)

// 2. Create screen model in tui/screens/your_screen.go
type YourScreenModel struct {
    // Your state here
}

func (m *YourScreenModel) Init() tea.Cmd {
    return nil
}

func (m *YourScreenModel) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    // Handle messages
}

func (m *YourScreenModel) View() string {
    // Render view
}

// 3. Register in tui/app.go Update() method
case ScreenYourNewScreen:
    var cmd tea.Cmd
    m.yourScreen, cmd = m.yourScreen.Update(msg)
    return m, cmd
```

### Screen Communication

Use `ScreenChangeMsg` for navigation:

```go
// Navigate to another screen
return m, func() tea.Msg {
    return ScreenChangeMsg{Screen: ScreenMounts}
}
```

## Styling Conventions

### Use the Shared Color Palette

```go
import "github.com/dtg01100/rclone-mount-sync/internal/tui/components"

// ✅ CORRECT - Use predefined colors
title := components.Styles.Title.Render("My Title")
error := components.Styles.Error.Render("Error occurred")

// ❌ WRONG - Don't create new colors
title := lipgloss.NewStyle().Foreground(lipgloss.Color("62")).Render("My Title")
```

### Available Styles

- `Styles.Title` - Main titles (bold, blue background)
- `Styles.Subtitle` - Subtitles (italic, muted)
- `Styles.Selected` / `Styles.Deselected` - List items
- `Styles.Error` / `Styles.Success` / `Styles.Warning` - Semantic feedback
- `Styles.Border` - Box borders
- `Styles.HelpText` - Help/tooltip text
- `Styles.Button` / `Styles.ButtonFocus` - Interactive buttons

## Form Handling with huh

For complex forms, use the `huh` library:

```go
import "github.com/charmbracelet/huh"

form := huh.NewForm(
    huh.NewGroup(
        huh.NewInput().
            Title("Name").
            Description("Enter a descriptive name").
            Validate(func(s string) error {
                if s == "" {
                    return fmt.Errorf("name is required")
                }
                return nil
            }),
        huh.NewSelect[string]().
            Title("Mode").
            Options(
                huh.NewOption("Full", "full"),
                huh.NewOption("Minimal", "minimal"),
            ),
    ),
).WithTheme(huh.ThemeCatppuccin())

// Run form and handle result
err := form.Run()
if err != nil {
    return m, func() tea.Msg { return err }
}
```

## Common Patterns

### Loading States

```go
// Show loading spinner
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg := msg.(type) {
    case LoadingMsg:
        m.loading = true
        return m, tea.Batch(
            yourAsyncCommand,
            func() tea.Msg { return LoadingDoneMsg{} },
        )
    case LoadingDoneMsg:
        m.loading = false
        return m, nil
    }
}
```

### Error Display

```go
// Use the Error style for consistency
if err != nil {
    return components.Styles.Error.Render(fmt.Sprintf("Error: %v", err))
}
```

### Help Text

Show keyboard shortcuts at bottom of screen:

```go
func (m *Model) View() string {
    help := components.Styles.HelpText.Render(
        "↑/k: up  ↓/j: down  Enter: select  q: back",
    )
    return content + "\n\n" + help
}
```

## Performance Considerations

### Avoid Expensive Operations in View()

```go
// ❌ WRONG - Don't do heavy work in View()
func (m *Model) View() string {
    data, _ := ioutil.ReadFile("large-file.txt")  // Bad!
    return string(data)
}

// ✅ CORRECT - Load data in Update(), cache in model
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    // Load data here, store in m.cachedData
}

func (m *Model) View() string {
    return m.cachedData  // Fast!
}
```

### Minimize Repaints

Only return commands when state actually changes:

```go
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    if msg == unchangedState {
        return m, nil  // No repaint needed
    }
    // ... handle change
}
```

## Keyboard Navigation

Standard key bindings across all screens:

| Key | Action | Context |
|-----|--------|---------|
| `↑` / `k` | Move up | Lists, forms |
| `↓` / `j` | Move down | Lists, forms |
| `Enter` | Select/Confirm | All interactions |
| `q` / `Esc` | Back/Cancel | Navigation |
| `n` | New/Create | List screens |
| `d` | Delete | List screens |
| `e` | Edit | List screens |
| `r` | Refresh | Data screens |
| `?` | Help | Global |

## Common Pitfalls

### ❌ Forgetting to Handle Messages

```go
// WRONG - Message ignored
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    return m, nil  // Always returns nil!
}

// CORRECT - Handle relevant messages
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg := msg.(type) {
    case tea.KeyMsg:
        return m.handleKeyPress(msg)
    case DataLoadedMsg:
        m.data = msg.Data
        return m, nil
    }
    return m, nil
}
```

### ❌ Not Preserving State

```go
// WRONG - Loses state on every update
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    newModel := NewModel()  // Resets everything!
    return newModel, nil
}

// CORRECT - Update existing model
func (m *Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    m.selectedIndex++  // Modify in place
    return m, nil
}
```
